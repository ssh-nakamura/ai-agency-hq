3. 3層コンテキストモデル

エージェントが起動時にどの情報を受け取るかを決定する、このプロジェクトの核となる設計。

┌─────────────────────────────────────────────────────────┐ │ Layer 1: CLAUDE.md（全員共通 / 自動注入 / ~55行） │ │ 組織ルール、指揮系統、禁止事項、エスカレーション │ │ CEO起動指示、サブエージェント起動指示 │ │ → 全エージェントのシステムプロンプトに自動注入 │ │ → 行数がそのままトークンコスト（全員に掛かる） │ ├─────────────────────────────────────────────────────────┤ │ Layer 2: MEMORY.md（個別 / 自動注入 / 200行制限） │ │ 先頭 ~10行: キャラクター設定（名前、口調、NG/OK例） │ │ 次の ~3行: 起動ルーティン（「agents/*.mdを読め」） │ │ 残り ~155行: ナレッジ蓄積（学び、判断基準、業務知識） │ │ → memory: project 設定で自動注入 │ │ → 200行超過分は切り捨て（先頭が生き残る設計） │ ├─────────────────────────────────────────────────────────┤ │ Layer 3: agents/*.md body（手動参照 / 非注入） │ │ 詳細な担当領域、フレームワーク、出力フォーマット │ │ 連携ルール、人間関係設定、フェーズ別行動指針 │ │ → bodyは自動注入されない（frontmatterのみ適用） │ │ → 起動時にReadツールで読む（1ターン消費） │ └─────────────────────────────────────────────────────────┘

5. ワークフロー

CEO起動フロー

通常モード（毎セッション）

自動: MEMORY.md 注入（キャラ + 知識） 自動: CLAUDE.md 注入（組織ルール） Read: docs/ceo-manual.md（起動指示） Read: docs/status.md（KPI + アクション + 収支） Check: 軽量セルフチェック 3項目 ☐ 2週間放置のアクションがないか ☐ 株主に上げる相談が溜まっていないか ☐ 前回の約束が完了しているか Action: 最優先アクションを実行 → 計 3ターンで作業開始

フルモード（週次 or 株主依頼時）

自動: MEMORY.md + CLAUDE.md Read: docs/ceo-manual.md Read: docs/status.md Read: docs/plan.md（事業計画 + ロードマップ） Run: python3 tools/validate-docs.py Check: 運用セルフチェック 6項目 Check: CEO改善セルフチェック 12項目 ☐ 使えるツール/MCPの見落としはないか ☐ 競合/市場に新しい動きはないか ☐ コスト削減・収益化前倒しの余地は ☐ エージェント教育不足の領域は … Output: 週次レポート or 戦略判断 → 計 6-7ターン

セッション終了フロー

1. HTML報告書を生成 → reports/session-YYYY-MM-DD.html → ブラウザで自動オープン （株主が見るのはこれだけ。サマリーカード + 実施内容 + 成果物 + 次回やること） 2. 内部ログを保存 → content/logs/YYYY-MM-DD.md （CEO参照用。簡潔に実施内容・決定事項・次回やること） 3. CEO MEMORY.mdを更新 （確定事項、失敗と学び） 4. docs/status.md を最新化 （完了タスク移動、新規タスク追加、KPI/収支変更） 5. トークン消費記録 （ccusage実行 → status.mdに記録）

エージェント呼び出しフロー

CEO判断: 今のタスクにどのエージェントが必要か？ 呼び出し: Task tool → subagent_type: "analyst" ※ "general-purpose" で呼ばない（キャラが読み込まれない） ※ プロンプトにキャラ設定を書かない（MEMORY.mdにある） サブエージェント起動時の自動処理: 1. CLAUDE.md が注入される 自動 2. MEMORY.md が注入される 自動 3. MEMORY.md内の指示に従い agents/*.md を Read 1ターン 4. 作業開始 事業別の担当: SaaS（ShieldMe等） → product-manager が主導 コンテンツ（ブログ等）→ writer が主導 動画（YouTube等） → video-creator が主導

エスカレーションフロー

レベル状況対応例
L1迷うが続行可能メモして後で報告表現の選択に迷った
L2方針不明中断してCEO判断仕様の解釈が分かれる
L3法的・倫理リスク即CEO報告→株主個人情報の取扱い
L4予算超過 ¥30,000以上CEO報告→株主承認API契約の追加

新事業の評価フロー

株主 or CEO: 「こういうのどう？」 → CEO: feasibility検討（analyst調査 + 担当PM or writer） → CEO → 株主: Go/NoGo提案（コスト・リスク・見込みを明記） → 承認 → status.mdのアクションに追加 基本哲学: 「AIで稼げることはなんでもやる実験室」 小さい実験はCEO判断でOK。コスト発生は株主承認。
