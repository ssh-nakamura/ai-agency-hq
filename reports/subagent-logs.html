## Claude Code Session Log Cleanup and Retention

Based on the official Claude Code documentation, here are the key findings:

### Automatic Cleanup Policy

**Yes, Claude Code does automatically delete session logs.** The cleanup is controlled by the `cleanupPeriodDays` setting in `settings.json`:

- **Default**: 30 days - Sessions inactive for longer than 30 days are automatically deleted at startup
- **Configurable**: You can customize this by setting `cleanupPeriodDays` to any number
- **Immediate deletion**: Setting `cleanupPeriodDays` to `0` immediately deletes all sessions at startup

### Configuration

The setting can be placed in `settings.json` at any scope level:
- User level: `~/.claude/settings.json`
- Project level: `.claude/settings.json`
- Local level: `.claude/settings.local.json`

Example configuration:
```json
{
"cleanupPeriodDays": 20
}
```

### Storage Details

- Session files (JSONL format) are stored in `~/.claude/projects/` directory
- Sessions are tied to directories and independent (each new session starts with a fresh context window)
- Configuration backups are automatically created and the **five most recent backups** are retained

### No Maximum Storage Limit

The documentation does not specify a maximum storage limit for sessions. The cleanup is based purely on **inactivity duration**, not on total storage size.

### Data Retention Summary

According to the **data retention policy** in the documentation:
- **Commercial users** (Team, Enterprise, API): Sessions cached locally for up to 30 days (configurable)
- **Consumer users**: Depends on your data privacy settings (5 years with data improvement opted in, 30 days otherwise)

**Note:** This refers to server-side retention. Locally stored session files follow the `cleanupPeriodDays` setting.
