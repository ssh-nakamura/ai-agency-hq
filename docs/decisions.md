# 意思決定ログ

> 重要な判断とその理由を時系列で記録する。
> 未来の自分（AI）が「なぜこうなったか」を追えるようにする。

## 更新ルール

### 誰が更新するか
- **CEOのみ**がこのファイルに追記する

### 記録フォーマット（必ずこの形式で）
```markdown
---

## YYYY-MM-DD / 判断の要約（1行）

**判断者**: 誰が判断したか
**決定**: 何を決めたか（1行）
**理由**: なぜそう決めたか（箇条書き2〜4行）
```

### 上限とアーカイブ
- **このファイルには直近20件だけ残す**
- 20件を超えたら古いものから `docs/archive/decisions-YYYY.md` に移動する
- 移動時、ファイル冒頭に「YYYY年の意思決定ログ」と記載する

### やってはいけないこと
- 過去の記録を編集・削除しない（ログは不変）
- フォーマットを変えない
- 些細な判断は記録しない（エージェントの呼び出し判断等は不要）

## 2026-02-14 / 組織構成をサブエージェント方式に決定

**判断者**: 株主 + CEO
**選択肢**:
- A: サブエージェント方式（CEOがメインエージェントとして他を呼び出す）
- B: Agent Teams方式（全員が対等に独立セッションで動く）

**決定**: A（サブエージェント方式）

**理由**:
- Agent Teamsはトークンコストが約7倍
- 現段階は売上ゼロ、コスト最小化が優先
- サブエージェント方式でもCEOが `claude --agent ceo` で起動すれば他エージェントを呼べる
- 事業が軌道に乗ったらBへの移行を検討

## 2026-02-14 / エージェント設計の再検討を決定

**判断者**: 株主
**決定**: 現行のサブエージェント方式（`--agent ceo`）を見直し、エージェント設計を一からやり直す

**理由**:
- `claude --agent` で起動するとTask toolが消えるバグ（Issue #13533）により、CEOから部下を呼べない
- Bash経由のネスト起動も不可（Claude Codeの仕様制約）
- Agent TeamsがMaxプラン定額内で使えることが判明し、コスト前提が変わった
- 候補: Agent Teams（Max加入時）or 素のClaude Code + CLAUDE.md（非Max時）
- 次回セッションでMaxプラン確認後に最終決定

## 2026-02-15 / 組織設計の全面改修（8エージェント体制）

**判断者**: 株主 + CEO
**決定**: 組織設計を全面改修。以下の方針を確定：

1. **事業スコープ**: 2本柱（SaaS + コンテンツ）ベース + 小さい実験はCEO判断でOK
2. **事業別担当**: PM→プロダクト/SaaS、writer→コンテンツ事業主導
3. **新事業フロー**: 株主アイデア → CEO feasibility → Go/NoGo → 実行
4. **哲学**: 「AIで稼げることはなんでもやる実験室」
5. **video-creator新設**: YouTube動画事業のためのエージェント追加（8人目）
6. **動画事業**: パイプライン設計まで今やる。ツール契約は別途株主承認

**理由**:
- ShieldMe単体に依存しない事業ポートフォリオが必要
- AI動画制作は2025〜2026年で急速に普及（クリエイター83%がAI利用）
- コストゼロで始められる部分（企画・台本・SEO）から着手
- 組織を「実験室」として設計することで、新事業への柔軟な対応力を確保

## 2026-02-15 / 部門間連携ルールの確定

**判断者**: 株主
**決定**: 部門長間の直接連携は「条件付きで許可」

- 作業レベルの連携（成果物受渡し、データ共有、レビュー依頼）→ OK
- 方針の相談 → OK、ただし結論はCEOに上げる
- 戦略変更 → 禁止。CEOに上げる

**理由**:
- 全部CEO経由だと非効率
- ただし戦略判断が分散するとカオスになるので、判断レベルはCEO経由を維持
