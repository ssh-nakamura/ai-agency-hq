# 意思決定ログ

> 重要な判断とその理由を時系列で記録する。
> 未来の自分（AI）が「なぜこうなったか」を追えるようにする。

## 更新ルール

### 誰が更新するか
- **CEOのみ**がこのファイルに追記する

### 記録フォーマット（必ずこの形式で）
```markdown
---

## YYYY-MM-DD / 判断の要約（1行）

**判断者**: 誰が判断したか
**決定**: 何を決めたか（1行）
**理由**: なぜそう決めたか（箇条書き2〜4行）
```

### 上限とアーカイブ
- **このファイルには直近20件だけ残す**
- 20件を超えたら古いものから `docs/archive/decisions-YYYY.md` に移動する
- 移動時、ファイル冒頭に「YYYY年の意思決定ログ」と記載する

### やってはいけないこと
- 過去の記録を編集・削除しない（ログは不変）
- フォーマットを変えない
- 些細な判断は記録しない（エージェントの呼び出し判断等は不要）

## 2026-02-14 / 組織構成をサブエージェント方式に決定

**判断者**: 株主 + CEO
**選択肢**:
- A: サブエージェント方式（CEOがメインエージェントとして他を呼び出す）
- B: Agent Teams方式（全員が対等に独立セッションで動く）

**決定**: A（サブエージェント方式）

**理由**:
- Agent Teamsはトークンコストが約7倍
- 現段階は売上ゼロ、コスト最小化が優先
- サブエージェント方式でもCEOが `claude --agent ceo` で起動すれば他エージェントを呼べる
- 事業が軌道に乗ったらBへの移行を検討

## 2026-02-14 / 永続メモリ（memory: project）を全エージェントに導入

**判断者**: 株主 + CEO
**理由**:
- セッション間で知識が引き継がれないと「会社」として機能しない
- 3層構造で設計：会社手引き（CLAUDE.md）/ 個人メモリ / 共有ドキュメント
- スコープは `project`（リポジトリ内に保存、チーム共有可能）

## 2026-02-14 / 組織構成を再編成（7エージェント体制）

**判断者**: 株主 + CEO
**変更内容**:
- `developer` → `product-manager`（仕様策定に専念、コードは別リポジトリ）
- `marketer` → `x-manager` + `writer` + `web-developer` に分割
