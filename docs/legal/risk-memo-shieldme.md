## リスク詳細分析

### リスク #1: AIカウンセリングの医療行為性

#### リスク内容

ShieldMeの「AIカウンセリング機能」は、ユーザーのメンタル状態を推測し、心理的サポートメッセージを提供します。この機能が以下の法律に抵触する可能性があります：

| 法律 | 条項 | リスク |
|------|------|--------|
| **医師法** | 第17条「医業とは診断を下し薬剤を投与すること」 | AIが「メンタル不調」と判定し、治療提案をすると医業に抵触する可能性 |
| **精神保健福祉法** | 第3条（基本理念）「精神疾患の発生予防と回復」 | 医学的根拠のない精神支援は法的グレーゾーン |
| **薬機法** | 第2条「医薬品販売の規制」 | メンタル薬の推奨は薬機法違反 |
| **カウンセリング法**（仮称） | 現在検討中 | 資格なしでのカウンセリング提供は将来規制される可能性 |

#### 最悪のケース

1. **ユーザーが精神疾患と誤診（AI判定）**
   → AIのメッセージに基づき、医療機関への受診を遅延
   → ユーザーの精神状態が悪化
   → 損害賠償請求（医療過誤に準ずる）

2. **規制当局（厚生労働省）からの指導**
   → 医師法違反の疑いで調査
   → サービス停止命令の可能性

#### 推奨する対応

**✅ 必須（MVP前に実装）**

1. **免責事項の明記（利用規約・サービス内表示）**
   ```
   「本機能は医療行為ではなく、医学的診断を提供しません。
   メンタルヘルスに関する懸念がある場合は、
   直ちに医療専門家（医師・心理士等）に相談してください。」
   ```
   - サービスのどこでもアクセス可能な場所に配置
   - ユーザーが初回利用時に同意する仕組み

2. **サービス内での表現の改善**
   - 「カウンセリング」→「メンタルサポート」に変更（医療的な印象軽減）
   - 「診断」という表現を使わない
   - 「参考情報です」という旨を常に表示

3. **医学的根拠の回避**
   - AI生成メッセージが「〇〇病の可能性がある」と述べないようにプロンプト調整
   - 生成メッセージのアウトプット検証（医療表現の自動フィルタリング）

4. **法律専門家への相談**
   - 医療法が専門の弁護士に「医業に該当するかどうか」の最終判定を得る
   - 厚生労働省への事前相談（任意）

**🟡 推奨（Phase 1から）**

5. **心理専門家の監修を検討**
   - AI生成メッセージが有害でないことを臨床心理士が確認
   - メッセージの医学的妥当性レビュー

6. **ユーザーリサーチ**
   - MVP利用ユーザーの反応調査
   - 「医療提供と誤解された」ケースの追跡

#### 関連法令・参考資料

### リスク #2: X API ToS違反

#### リスク内容

ShieldMeはX APIを使用してユーザーデータを取得します。X（Twitter）のAPI利用規約にはいくつかの制限があり、これに違反するとAPI取消しのリスクがあります。

#### X API利用規約で特に注意すべき条項

| 条項 | 説明 | ShieldMeへの影響 |
|------|------|-----------------|
| **暴力・ハラスメント検知** | 「暴力やハラスメント検知用途でのAPI利用は申請が必要」 | ShieldMeは誹謗中傷検知なので申請対象の可能性 |
| **個人情報の保護** | 「取得した個人情報を第三者に提供してはいけない」 | Anthropic（Claude API）への送信が違反に該当する可能性 |
| **キャッシング制限** | 「取得データを無制限に保存してはいけない」 | 過去3ヶ月のキャッシュは規約範囲内か要確認 |
| **表示要件** | 「Xとの連携を明示する」 | 利用規約に記載し、サービスUIで表示必須 |

#### 最悪のケース

1. **API利用申請の却下**
   → ShieldMeが機能しなくなる（データ取得不可）
   → 有料ユーザーへの返金義務発生

2. **API利用の無予告中止**
   → サービス継続不可
   → ユーザーデータの処理方法に困窮

#### 推奨する対応

**✅ 必須（MVP公開前に実施）**

1. **X APIの申請要件の確認**
   ```
   - X APIダッシュボードで現在の「API利用目的」を確認
   - 「ハラスメント検知」カテゴリでの追加申請が必要かどうかを確認
   - X APIドキュメントで最新の制限事項を確認
   ```
   - 参考: https://developer.twitter.com/

2. **プライバシーポリシーでの明記**
   - 「本サービスはX APIを利用しています」と明記
   - Xの利用規約への外部リンク掲載

3. **データ第三者提供の透明化**
   - プライバシーポリシーに「Anthropic（Claude API）にテキストデータを送信する」と明記
   - X APIからの取得データをそのまま第三者に提供しないよう設計確認

4. **定期的な規約監視**
   - X Developer Forumのアナウンスをモニター
   - 規約変更時の影響評価

**🟡 推奨（ベータ版展開時）**

5. **X側への事前相談**
   - X API申請時に「誹謗中傷検知用途」であることを明記
   - 許可を得た上でベータ版公開

### リスク #3: AI判定の誤判定リスク（名誉毀損）

#### リスク内容

ShieldMeのAI判定が誤った場合（見落とし・過検知）、以下の法的責任が問題になります：

| 判定エラーの種類 | 法的リスク | 深刻度 |
|-----------------|-----------|--------|
| **過検知（誤検知）** | 誹謗中傷ではない投稿を「誹謗中傷」と判定 | 投稿者から「名誉毀損（名誉棄損）」として賠償請求される可能性 |
| **見落とし** | 本来的に誹謗中傷である投稿を見落とし | ユーザーから「サービスの欠陥」として損害賠償請求される可能性 |
| **判定に基づく行動** | ユーザーが「ShieldMeの判定を信用して」違法投稿（名誉毀損）を行った場合 | 当社が「違法行為の助長」として責任を問われる可能性 |

#### 最悪のケース

```
シナリオ: ユーザーAが誹謗中傷投稿を受けた
       → ShieldMeが「強い非難」と判定
       → 実際には誹謗中傷ではない投稿だった
       → ユーザーが「ShieldMeの判定」に基づいて
         投稿者を誣告罪で告発した
       → 告発は却下（実際には誹謗中傷ではなかったため）
       → 投稿者が当社を賠償請求

責任所在: 当社の誤判定 → 損害賠償請求（¥100万程度の可能性）
```

#### 推奨する対応

**✅ 必須（利用規約で対応）**

1. **免責事項の明記**（利用規約・プライバシーポリシーで既記載）
   ```
   「本サービスの判定結果は、参考情報に過ぎず、
   完全な正確性は保証されません。
   判定結果に基づく行動は、ユーザー自身の判断です。」
   ```

2. **判定の「参考値」としての表示**
   - 判定スコアを「数値」で表示（「完全に誹謗中傷」でなく「スコア 0.87」）
   - 「AIの判定結果です。最終判断はご自身で」という注記

3. **フィードバック機能の実装**
   - ユーザーが「判定が間違っている」とフィードバック可能な仕組み
   - 誤判定データの収集・改善

**🟡 推奨（Phase 1から）**

4. **AI判定の透明性向上**
   - 判定理由の説明（「〇〇という表現が含まれているため」等）
   - ユーザーが判定に異議を唱えるプロセスの整備

5. **定期的な精度監査**
   - 判定精度の測定（Precision / Recall）
   - 月次での精度レポート作成

6. **保険への加入検討**
   - 賠償責任保険（Professional Indemnity Insurance）の加入
   - 弁護士費用保険

### リスク #4: 個人情報の過度な処理

#### リスク内容

ShieldMeはAIを用いて、ユーザーの「心理状態」「関心事」「社会的属性」等を推測します。これは「プロファイリング」に該当し、個人情報保護法の規制対象です。

| 処理内容 | 法的性質 | リスク |
|--------|---------|--------|
| 心理状態の推測 | センシティブ情報の処理 | 個人情報保護法第16条「本人の同意」が必須 |
| 関心事・トピックの推測 | プロファイリング | 処理内容・目的の透明性が必須 |
| 社会的属性（推定年代等）の推測 | 差別につながる可能性 | 利用目的の限定が必須 |

#### 最悪のケース

1. **個人情報保護委員会からの指導**
   - 同意なし、または適切な説明なしの個人情報処理
   - 改善命令、氏名公表、罰金（最大¥100万）

2. **ユーザーからの個人情報開示請求**
   - 「自分の心理状態をどのように推測されているのか知りたい」
   - 当社が対応不可能な場合、法的問題化

#### 推奨する対応

**✅ 必須（MVP前に実装）**

1. **同意管理の強化**
   - AI分析（カウンセリング機能）の利用に対する**明示的な同意**
   - 「心理状態の推測を許可しますか？」という選択肢

2. **透明性の向上**
   - プライバシーポリシーに「どのような推測をするのか」を記載
   - 推測データの閲覧機能（ユーザーが「AIが推測している自分の情報」を見られる仕組み）

3. **データの個人化解除**
   - 推測データを「個人を特定できない形」に変換
   - 学習用データとしての利用は個人情報から離別

**🟡 推奨（Phase 1から）**

4. **プライバシー影響評価（PIA）の実施**
   - 個人情報保護方針に基づき、AI処理のリスク分析
   - 外部の専門家による監査

### リスク #5: 決済・返金トラブル

#### リスク内容

ShieldMeは月額サブスクリプション（¥980, ¥4,980）で課金されます。決済トラブルのリスク：

| リスク | 法的根拠 | 対応 |
|-------|---------|------|
| **返金拒否による消費者トラブル** | 特定商取引法第9条（返金ルール） | 利用規約に「14日返金保証」を記載済み |
| **継続課金の停止手続きの複雑さ** | 消費者契約法第9条の2（継続的役務契約） | 「1クリックで解除できる」仕様にする |
| **クーリングオフ権** | 特定商取引法第8条 | オンライン契約はクーリングオフ対象外（原則） |

#### 推奨する対応

**✅ 必須（決済実装時）**

1. **特定商取引法への対応**
   - 「販売者の名称」「住所」「連絡先」をサービス内に表示
   - 返金ポリシーの明記（「14日以内の返金」等）

2. **継続課金停止の簡易化**
   - サービス内から「解約」を1クリックで実行できる仕組み
   - メールでの解約手続き廃止（サービス内のみ）

3. **決済ログの保存**
   - 返金トラブル時に備えて、決済記録を最低7年保存

### リスク #6: AI学習データの著作権

#### リスク内容

ShieldMeが投稿テキストを処理する際、Anthropicの Claude APIを使用しています。これらのデータが「著作物」の場合、著作権侵害のリスクがあります。

| 問題 | リスク | 対応 |
|------|--------|------|
| **X投稿の著作権** | ユーザーがX投稿した文章は、投稿者に著作権がある | ShieldMeが「学習データ」として使用すると著作権侵害の可能性 |
| **Claude APIでのデータ利用** | Anthropicがユーザーデータを「モデル改善」に利用する場合 | ユーザーの同意が必須 |
| **AI生成コンテンツの権利** | ShieldMeが生成するカウンセリングメッセージの著作権 | 当社またはAnthropicのどちらに帰属するか |

#### 推奨する対応

**✅ 必須（MVP前に確認）**

1. **Anthropicの利用規約確認**
   - Claude APIの「データ利用ポリシー」を確認
   - ユーザーデータをモデル改善に使用するかどうか確認

2. **プライバシーポリシーでの記載**
   - 「Anthropicに送信されたデータ利用について」を記載
   - Anthropicのプライバシーポリシーへのリンク掲載

3. **X投稿データの取扱い**
   - X APIから取得した投稿は「参考値」として取扱い
   - 学習データとしての無期限保存は避ける

**🟡 推奨（規模拡大時）**

4. **ファインチューニングの検討**
   - 将来的に「ShieldMe専用の学習モデル」を作成する場合、著作権ライセンス取得が必須

## 最後に（所見）

ShieldMeは「AIで被害者を守る」という高い倫理的目的を持つサービスです。ただし、以下の点に留意する必要があります：

1. **医療行為との線引きが最も重要** — 「心理サポート」と「医療」の境界線は、日本の法制度でもグレーゾーンです。MVP段階では、免責事項の強化とメッセージプロンプトの調整で対応し、将来的には医療専門家の監修導入を検討してください。

2. **X APIの継続利用が事業の根幹** — APIが利用中止になると、ShieldMeは機能しません。X APIダッシュボードでの申請要件確認は早期に実施してください。

3. **AI判定の完全性を保証しない** — これが利用規約で適切に記載されれば、誤判定による賠償請求リスクは軽減されます。

4. **個人情報保護が競合優位性** — コンプライアンスを徹底することで、「信頼できるサービス」として差別化できます。
