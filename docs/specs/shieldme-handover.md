## 1. プロダクト概要

**名前**: ShieldMe（シールドミー）

**ワンライナー**: 「AIがエゴサを代行し、応援を届け、誹謗中傷から心を守るSaaS」

**概要**: 個人クリエイター・インフルエンサーのためのAI誹謗中傷検知SaaS。SNSのエゴサをAIが代行し、ポジティブなフィードバックを前面に表示する一方、ネガティブなコメントは要約・抽象化して心理的負担を軽減する。

**ポジショニング**: 企業向け月5万円以上が当たり前の市場に、個人向け月980円で参入する「価格破壊ポジション」。

## 2. 解決する課題とターゲット

### 課題の背景

- SNSでの誹謗中傷は個人クリエイターの深刻な悩み（プロ野球選手会もシステム導入）
- デジタルリスク対策市場は年平均20%成長（2024年度139億円規模、MM総研）
- 既存サービスは月額5万円以上の企業向けが中心で、個人には手が届かない
- エゴサは精神的負荷が高く、ポジティブなコメントも誹謗中傷に埋もれて見落とす

### ペインポイント

1. **エゴサの精神的負荷**: 自分で見ると直接傷つく
2. **ポジティブの見落とし**: 誹謗中傷に目が行き、応援コメントを見逃す
3. **価格の壁**: 既存サービスは月5万円〜で個人には高額
4. **批判と誹謗中傷の境界**: 何が「許容すべき批判」で何が「対処すべき誹謗中傷」か判断できない

## 3. MVP機能仕様

### Must Have（MVP必須機能）

#### 3.1 キーワード登録

| 項目 | 詳細 |
|------|------|
| ユーザーストーリー | VTuberとして、自分のハンドル名とニックネームを登録したい。全てのSNS投稿を正確に捕捉するため |
| 受け入れ条件 | 3つ以上のキーワード登録可能、リアルタイム検証（検索結果プレビュー） |
| RICEスコア | 90 |

#### 3.2 X投稿検索

| 項目 | 詳細 |
|------|------|
| ユーザーストーリー | インフルエンサーとして、過去7日間の自分への言及を取得したい。手動エゴサの負担を減らすため |
| 受け入れ条件 | X API v2で100件まで取得、検索クエリを自動生成、自分の投稿を除外 |
| RICEスコア | 95 |

#### 3.3 AI分類（ポジティブ/ネガティブ）

| 項目 | 詳細 |
|------|------|
| ユーザーストーリー | クリエイターとして、AIに自動で感情分類してほしい。全てを自分で読まずに済むため |
| 受け入れ条件 | POSITIVE/NEUTRAL/NEGATIVEの3分類、信頼度スコア付き、LLMベースで文脈を理解 |
| RICEスコア | 100 |

#### 3.4 ポジティブダッシュボード

| 項目 | 詳細 |
|------|------|
| ユーザーストーリー | 配信者として、応援メッセージを前面に表示してほしい。モチベーションを保つため |
| 受け入れ条件 | ポジティブ投稿を上位表示、いいね数・RT数でソート可能、全体の感情割合を可視化 |
| RICEスコア | 85 |

#### 3.5 ネガティブ要約表示

| 項目 | 詳細 |
|------|------|
| ユーザーストーリー | クリエイターとして、誹謗中傷の原文は見たくない。趣旨だけ知りたい |
| 受け入れ条件 | 原文を隠す、AI要約で抽象化、深刻度ラベル（高/中/低）、必要時のみ原文表示 |
| RICEスコア | 100 |

#### 3.6 AIカウンセリング

| 項目 | 詳細 |
|------|------|
| ユーザーストーリー | 誹謗中傷に悩む配信者として、AIに相談したい。気持ちを整理するため |
| 受け入れ条件 | チャット形式で対話、共感的なトーン、専門家への相談を適宜推奨、法的助言はしない |
| RICEスコア | 75 |
| **法務上の注意** | **「カウンセリング」ではなく「メンタルサポート」と表記すること（医療行為性の回避）** |

### Phase 2で追加（MVP外）

| 機能 | 理由 |
|------|------|
| ユーザー認証・ログイン | MVPはStreamlitのセッション管理で代用 |
| 定期監視（1日1回自動） | MVPは手動実行 |
| 証拠保存・エクスポート（PDF/CSV） | 法的対応にはPhase 2で必要 |
| プッシュ通知（脅迫レベルのアラート） | Phase 2でメール通知から実装 |
| 決済機能（Stripe連携） | ベータテスト後に実装 |

## 4. ユーザーフロー

### メインフロー（初回利用）

```
1. ランディングページ訪問
   ↓
2. 「無料で試す」ボタン
   ↓
3. キーワード入力画面
   - Xハンドル（@example）
   - 本名（オプション）
   - ニックネーム（オプション）
   ↓
4. 「検索開始」ボタン
   ↓
5. 処理中画面（プログレスバー表示）
   - X投稿を取得中...
   - AIが分類中...（100件なら30秒程度）
   ↓
6. ポジティブダッシュボード表示
   - サマリー（ポジティブ○件、ネガティブ○件）
   - 応援メッセージ一覧
   - 下部に「ネガティブレポートを見る」リンク
   ↓
7. ネガティブレポート（オプション）
   - 要約のみ表示
   - 各要約に「AIに相談する」ボタン
   ↓
8. AIメンタルサポート（オプション）
   - チャット画面
```

### セカンダリフロー（リピート利用）

```
1. 再訪問
   ↓
2. キーワード入力（前回の値を記憶）
   ↓
3. 期間選択（過去7日間 / 過去24時間）
   ↓
4. 検索 → ダッシュボード
```

## 5. 技術要件

### アーキテクチャ

```
┌─────────────┐
│ ユーザー     │
│ (ブラウザ)   │
└──────┬──────┘
       │ HTTPS
       ▼
┌─────────────────────┐
│ Streamlit App       │
│ (Python)            │
│ ┌─────────────────┐ │
│ │ ポジティブ      │ │
│ │ ダッシュボード  │ │
│ ├─────────────────┤ │
│ │ ネガティブ      │ │
│ │ レポート        │ │
│ ├─────────────────┤ │
│ │ AI              │ │
│ │ メンタルサポート│ │
│ └─────────────────┘ │
└────┬────────┬───────┘
     │        │
     ▼        ▼
┌─────────┐ ┌─────────┐
│ X API v2│ │Claude API│
│ (検索)  │ │(Haiku/  │
│         │ │ Sonnet) │
└─────────┘ └─────────┘
```

### 技術スタック

| レイヤー | 技術 | 選定理由 | 代替案 |
|----------|------|----------|--------|
| **フロントエンド** | Streamlit | Pythonで完結、リッチUIが簡単、プロトタイプに最適 | Next.js（Phase 2で検討） |
| **バックエンド** | Python 3.11+ | Streamlitと統合、AI/NLP系ライブラリが豊富 | — |
| **SNS検索** | X API v2（Basic Tier） | 月$100で10,000リクエスト、最新APIでサポート安定 | スクレイピング（規約違反リスク） |
| **AI分類** | Claude API（Haiku 4.5） | 1件$0.0005と安価、文脈理解が強い、速度と精度のバランス良 | GPT-4o-mini（コスト同等） |
| **AIメンタルサポート** | Claude API（Sonnet 4.5） | 共感力が必要なタスク、Haikuより文章が自然 | GPT-4o（やや高額） |
| **ホスティング** | Streamlit Cloud（Community） | 無料、デプロイ簡単、HTTPS標準 | Vercel（Phase 2移行時） |
| **セッション管理** | Streamlit session_state | MVPはメモリで十分、DBレス運用 | Supabase（Phase 2） |
| **認証** | なし（MVPは認証なし） | 機能検証に集中 | Clerk / Auth0（Phase 2） |

### 外部サービス・API

| サービス | プラン | 月額コスト | 用途 |
|---------|--------|-----------|------|
| X API v2 | Basic | $100（≒¥15,000） | 投稿検索（10,000リクエスト/月） |
| Claude API | 従量課金 | 変動（¥5,000〜15,000想定） | 分類・要約・メンタルサポート |
| Streamlit Cloud | Community | ¥0 | ホスティング |
| **合計** | | **≒¥20,000〜30,000/月** | |

### 性能要件

| 項目 | 目標値 | 根拠 |
|------|--------|------|
| 検索処理時間 | 100件を30秒以内 | X API: 5秒 + Claude API（バッチ）: 20秒 + UI描画: 5秒 |
| ダッシュボード表示 | 2秒以内 | Streamlitのキャッシュ活用 |
| メンタルサポート応答 | 5秒以内/往復 | Claude Sonnetのストリーミング応答 |
| 同時ユーザー数 | 10人（MVP） | Streamlit Community制限。Phase 2でスケール |

### セキュリティ要件

| 項目 | 要件 | 実装方法 |
|------|------|----------|
| API Key管理 | 環境変数で秘匿 | Streamlit Secrets（.streamlit/secrets.toml） |
| HTTPS通信 | 全通信を暗号化 | Streamlit Cloud標準機能 |
| ユーザーデータ | セッション終了時に破棄 | session_stateのみ使用（DB保存なし） |
| 原文表示制限 | デフォルトで非表示 | 「表示する」ボタンで明示的に開示 |
| 法的リスク免責 | 利用規約で明示 | 後述の法務要件を参照 |

### 可用性要件

| 項目 | 目標 | MVP対応 |
|------|------|---------|
| 稼働率 | 95%以上 | Streamlit Cloudに依存（SLA未保証） |
| エラー時の動作 | エラーメッセージ表示 | try-except + ユーザーフレンドリーなエラー文 |
| ダウンタイム | 週1回のメンテ許容 | MVPは小規模のため影響小 |

## 6. 競合分析

### 市場構造

デジタルリスク対策サービス市場は「検知系」と「対策系」に二分。ShieldMeは検知系。

| 指標 | 数値 |
|------|------|
| 市場規模（2024年度） | 約139億円（MM総研） |
| 市場規模（2021年度） | 約77億円 |
| 年平均成長率 | 約20%+ |
| 検知系サービス | 35.3億円（2021年度） |
| 対策系サービス | 42.1億円（2021年度） |

### 市場拡大の背景

- SNS利用者の増加と炎上リスクの高まり
- 2024年のプロ野球選手会による誹謗中傷検出システム導入（社会的認知の向上）
- 企業のデジタルリスク意識の高まり（約6〜7割の企業が支出増）
- 法改正（侮辱罪の厳罰化等）による対策ニーズの顕在化

## 10. 法務要件

### 10.1 法的リスクの優先順位

| # | リスク領域 | 深刻度 | 発生確率 | 対応タイミング |
|---|-----------|--------|---------|-------------|
| 1 | AIメンタルサポートの医療行為性 | **Critical** | 中 | **MVP前に必須** |
| 2 | X API利用規約違反 | **High** | 高 | **MVP前に必須** |
| 3 | AI判定の誤判定（名誉毀損） | **High** | 中 | 利用規約で対応 |
| 4 | 個人情報の過度な処理 | **Medium** | 中 | プライバシー設計 |
| 5 | 決済・返金トラブル | **Medium** | 低 | 決済実装時 |
| 6 | AI学習データの著作権 | **Medium** | 低 | Anthropic規約確認 |

### 10.2 リスク#1: 医療行為性（Critical）

**問題**: 「AIカウンセリング機能」が医師法第17条（医業の禁止）に抵触する可能性。

**MVP前に必須の対応**:

1. **名称変更**: 「カウンセリング」→「メンタルサポート」（医療的印象を軽減）
2. **免責事項の常時表示**:
   ```
   「本機能は医療行為ではなく、医学的診断を提供しません。
   メンタルヘルスに関する懸念がある場合は、
   直ちに医療専門家（医師・心理士等）に相談してください。」
   ```
3. **プロンプト調整**: AIが「〇〇病の可能性がある」と述べないよう制御
4. **「診断」という表現を絶対に使わない**
5. **初回利用時の同意取得**

**最悪のケース**: ユーザーがAI判定を信じて医療機関受診を遅延 → 精神状態悪化 → 損害賠償請求、厚生労働省からの指導

### 10.3 リスク#2: X API利用規約（High）

**注意すべき条項**:

| 条項 | ShieldMeへの影響 |
|------|-----------------|
| 暴力・ハラスメント検知用途は申請が必要 | 誹謗中傷検知なので**申請対象の可能性** |
| 取得した個人情報を第三者に提供してはいけない | Claude APIへの送信が**違反に該当する可能性** |
| 取得データを無制限に保存してはいけない | キャッシュ期間の制限確認が必要 |
| Xとの連携を明示する | UIに表示必須 |

**MVP前に必須**: X APIダッシュボードで利用目的の申請確認。「ハラスメント検知」カテゴリでの追加申請が必要かどうか確認する。

### 10.4 リスク#3: 誤判定（High）

**実装で対応すべきこと**:

1. 判定結果に「参考情報」の注記を常に表示
2. 信頼度スコアを数値で表示（「完全に誹謗中傷」ではなく「スコア 0.87」）
3. ユーザーが「判定が間違っている」とフィードバック可能な仕組み
4. 利用規約で免責を明記

### 10.5 利用規約（草案あり）

以下の利用規約草案がHQで作成済み。開発リポジトリにコピーして使用可能。

**主要条項**:
- 第1条: サービスの定義と規約適用
- 第2条: ユーザーの義務（禁止行為、アカウント管理）
- 第3条: AI判定と免責事項（判定は推測値、完全な正確性は非保証）
- 第4条: AIメンタルサポート機能（医療行為ではない旨の明記）
- 第5条: データの取扱い
- 第6条: 料金・返金（14日以内の返金保証）
- 第7条: 責任制限（賠償上限は当月サービス月額料金）
- 第8条: 知的財産権
- 第9条: サービス中断・終了（終了時30日前通知）
- 第10条: 準拠法（日本法）、管轄裁判所（東京地方裁判所）

**注意**: この利用規約はAI作成の草案。法律専門家による最終レビューが**必須**。

### 10.6 プライバシーポリシー（草案あり）

**取得する個人情報**:

## 13. リスク分析

### プロダクトリスク

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|---------|------|
| LLMのAPI費用がユーザー単価を上回る | 高 | 中 | 実測でコスト検証 → Batch API活用 → 価格プラン見直し |
| 誤検知（ポジティブを誹謗中傷と判定） | 高 | 中 | 信頼度スコア表示 + ユーザー修正機能 + 免責条項 |
| X APIの仕様変更・価格改定 | 中 | 低 | 代替手段の調査 + 他SNSへの展開準備 |
| 大手企業が個人向けプランを投入 | 中 | 中 | UXでの差別化（ポジティブ表示+メンタルサポート）+ 先行者利益 |
| 法的トラブル（誤検知による名誉毀損） | 高 | 低 | 利用規約で免責 + 「参考情報」の明示 + 法務レビュー |

### 事業リスク

| リスク | 対策 |
|--------|------|
| APIコスト > 収益 | 収支計画で損益分岐点を明確化。Phase 1で実測 |
| X API利用停止（規約違反で） | 事前にX Developer Portalで申請確認。代替検索手段の調査 |
| 競合の価格破壊 | ポジティブ・ファースト設計 + メンタルサポートは独自価値 |
| ユーザーが集まらない | VTuber・配信者コミュニティへの集中マーケティング |

### 法的リスク（再掲・重要度順）

1. **Critical**: AIメンタルサポートが医療行為と判断される → 免責事項 + 名称変更で対応
2. **High**: X API利用規約違反 → 事前申請確認
3. **High**: AI判定の誤りによる名誉毀損請求 → 利用規約 + 「参考情報」表記
