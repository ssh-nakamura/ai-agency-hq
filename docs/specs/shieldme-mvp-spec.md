## 作成日: 2026-02-15

## 1. 概要

**ShieldMe（シールドミー）**は、個人クリエイター・インフルエンサーのためのAI誹謗中傷検知SaaSです。SNSのエゴサをAIが代行し、ポジティブなフィードバックを前面に表示する一方、ネガティブなコメントは要約・抽象化して心理的負担を軽減します。

**ワンライナー:** 「AIがエゴサを代行し、応援を届け、誹謗中傷から心を守るSaaS」

---

## 2. 解決する課題

### 課題の背景

- SNSでの誹謗中傷は個人クリエイターの深刻な悩み（プロ野球選手会もシステム導入）
- デジタルリスク対策市場は年平均20%成長（2024年度139億円規模、MM総研）
- 既存サービスは月額5万円以上の企業向けが中心で、個人には手が届かない
- エゴサは精神的負荷が高く、ポジティブなコメントも誹謗中傷に埋もれて見落とす

### ペインポイント

1. **エゴサの精神的負荷**: 自分で見ると直接傷つく
2. **ポジティブの見落とし**: 誹謗中傷に目が行き、応援コメントを見逃す
3. **価格の壁**: 既存サービスは月5万円〜で個人には高額
4. **批判と誹謗中傷の境界**: 何が「許容すべき批判」で何が「対処すべき誹謗中傷」か判断できない

## 3. 主要機能

### MVP（Must Have）

| 機能 | ユーザーストーリー | 受け入れ条件 | RICEスコア |
|------|-------------------|-------------|-----------|
| キーワード登録 | VTuberとして、自分のハンドル名とニックネームを登録したい。全てのSNS投稿を正確に捕捉するため | ・3つ以上のキーワード登録可能<br>・リアルタイム検証（検索結果プレビュー） | 90 |
| X投稿検索 | インフルエンサーとして、過去7日間の自分への言及を取得したい。手動エゴサの負担を減らすため | ・X API v2で100件まで取得<br>・検索クエリを自動生成<br>・自分の投稿を除外 | 95 |
| AI分類（ポジティブ/ネガティブ） | クリエイターとして、AIに自動で感情分類してほしい。全てを自分で読まずに済むため | ・POSITIVE/NEUTRAL/NEGATIVEの3分類<br>・信頼度スコア付き<br>・LLMベースで文脈を理解 | 100 |
| ポジティブダッシュボード | 配信者として、応援メッセージを前面に表示してほしい。モチベーションを保つため | ・ポジティブ投稿を上位表示<br>・いいね数・RT数でソート可能<br>・全体の感情割合を可視化 | 85 |
| ネガティブ要約表示 | クリエイターとして、誹謗中傷の原文は見たくない。趣旨だけ知りたい | ・原文を隠す<br>・AI要約で抽象化<br>・深刻度ラベル（高/中/低）<br>・必要時のみ原文表示 | 100 |
| AIカウンセリング | 誹謗中傷に悩む配信者として、AIに相談したい。気持ちを整理するため | ・チャット形式で対話<br>・共感的なトーン<br>・専門家への相談を適宜推奨<br>・法的助言はしない | 75 |

**RICE算出例（AI分類）:**
- Reach: 100%（全ユーザーが使う）
- Impact: 3（大 - コア価値）
- Confidence: 100%
- Effort: 3人日
- **RICE = (100 × 3 × 1.0) / 3 = 100**

### Phase 2（Should Have）

| 機能 | 理由 |
|------|------|
| ユーザー認証・ログイン | MVPはStreamlitのセッション管理で代用。本番では必須 |
| 定期監視（1日1回自動） | MVPは手動実行。自動化は運用コスト次第 |
| 証拠保存・エクスポート（PDF/CSV） | 法的対応には必要だが、MVPでは画面スクショで代用 |
| プッシュ通知（脅迫レベルのアラート） | 重要だがMVPスコープ外。メール通知で代用可能 |
| 決済機能（Stripe連携） | 収益化には必須だが、MVP検証フェーズでは不要 |

|------|-----------|

| チーム機能（事務所向け） | Phase 3。個人での成功が先 |
| 弁護士連携 | Phase 4。提携先確保が必要 |
| 多言語対応 | Won't Have（国内市場で十分） |

## 4. ユーザーフロー

### メインフロー（初回利用）

```
1. ランディングページ訪問
   ↓
2. 「無料で試す」ボタン
   ↓
3. キーワード入力画面
   - Xハンドル（@example）
   - 本名（オプション）
   - ニックネーム（オプション）
   ↓
4. 「検索開始」ボタン
   ↓
5. 処理中画面（プログレスバー表示）
   - X投稿を取得中...
   - AIが分類中...（100件なら30秒程度）
   ↓
6. ポジティブダッシュボード表示
   - サマリー（ポジティブ○件、ネガティブ○件）
   - 応援メッセージ一覧
   - 下部に「ネガティブレポートを見る」リンク
   ↓
7. ネガティブレポート（オプション）
   - 要約のみ表示
   - 各要約に「AIに相談する」ボタン
   ↓
8. AIカウンセリング（オプション）
   - チャット画面
```

### セカンダリフロー（リピート利用）

```
1. 再訪問
   ↓
2. キーワード入力（前回の値を記憶）
   ↓
3. 期間選択（過去7日間 / 過去24時間）
   ↓
4. 検索 → ダッシュボード
```

## 5. 技術要件

### アーキテクチャ概要

```
┌─────────────┐
│ ユーザー     │
│ (ブラウザ)   │
└──────┬──────┘
       │ HTTPS
       ▼
┌─────────────────────┐
│ Streamlit App       │
│ (Python)            │
│ ┌─────────────────┐ │
│ │ ポジティブ      │ │
│ │ ダッシュボード  │ │
│ ├─────────────────┤ │
│ │ ネガティブ      │ │
│ │ レポート        │ │
│ ├─────────────────┤ │
│ │ AI              │ │
│ │ カウンセリング  │ │
│ └─────────────────┘ │
└────┬────────┬───────┘
     │        │
     ▼        ▼
┌─────────┐ ┌─────────┐
│ X API v2│ │Claude API│
│ (検索)  │ │(Haiku/  │
│         │ │ Sonnet) │
└─────────┘ └─────────┘
```

### 技術スタック

| レイヤー | 技術 | 選定理由 | 代替案 |
|----------|------|----------|--------|
| **フロントエンド** | Streamlit | ・Pythonで完結（学習コスト低）<br>・リッチUIが簡単<br>・プロトタイプに最適 | Next.js（Phase 2で検討） |
| **バックエンド** | Python 3.11+ | ・Streamlitと統合<br>・AI/NLP系ライブラリが豊富 | - |
| **SNS検索** | X API v2（Basic Tier） | ・月$100で10,000リクエスト<br>・最新APIでサポート安定 | スクレイピング（規約違反リスク） |
| **AI分類** | Claude API（Haiku 4.5） | ・1件$0.0005と安価<br>・文脈理解が強い<br>・速度と精度のバランス良 | GPT-4o-mini（コスト同等） |
| **AIカウンセリング** | Claude API（Sonnet 4.5） | ・共感力が必要なタスク<br>・Haikuより文章が自然 | GPT-4o（やや高額） |
| **ホスティング** | Streamlit Cloud（Community） | ・無料<br>・デプロイが簡単<br>・HTTPS標準 | Vercel（Phase 2移行時） |
| **セッション管理** | Streamlit session_state | ・MVPはメモリで十分<br>・DBレス運用 | Supabase（Phase 2） |
| **認証** | なし（MVPは認証なし） | ・機能検証に集中 | Clerk / Auth0（Phase 2） |

### 外部サービス・API

| サービス | プラン | 月額コスト | 用途 |
|---------|--------|-----------|------|
| X API v2 | Basic | $100（≒¥15,000） | 投稿検索（10,000リクエスト/月） |
| Claude API | 従量課金 | 変動（¥5,000〜15,000想定） | 分類・要約・カウンセリング |
| Streamlit Cloud | Community | ¥0 | ホスティング |
| **合計** | | **≒¥20,000〜30,000/月** | Phase 1想定 |

### 性能要件

| 項目 | 目標値 | 根拠 |
|------|--------|------|
| 検索処理時間 | 100件を30秒以内 | ・X API: 5秒<br>・Claude API（バッチ）: 20秒<br>・UI描画: 5秒 |
| ダッシュボード表示 | 2秒以内 | Streamlitのキャッシュ活用 |
| カウンセリング応答 | 5秒以内/往復 | Claude Sonnetのストリーミング応答 |
| 同時ユーザー数 | 10人（MVP） | Streamlit Community制限。Phase 2でスケール |

### セキュリティ要件

| 項目 | 要件 | 実装方法 |
|------|------|----------|
| API Key管理 | 環境変数で秘匿 | Streamlit Secrets（.streamlit/secrets.toml） |
| HTTPS通信 | 全通信を暗号化 | Streamlit Cloud標準機能 |
| ユーザーデータ | セッション終了時に破棄 | session_stateのみ使用（DB保存なし） |
| 原文表示制限 | デフォルトで非表示 | 「表示する」ボタンで明示的に開示 |
| 法的リスク免責 | 利用規約で明示 | legalが別途作成 |

### 可用性要件

| 項目 | 目標 | MVP対応 |
|------|------|---------|
| 稼働率 | 95%以上 | Streamlit Cloudに依存（SLA未保証） |
| エラー時の動作 | エラーメッセージ表示 | try-except + ユーザーフレンドリーなエラー文 |
| ダウンタイム | 週1回のメンテ許容 | MVPは小規模のため影響小 |

## 11. MVP開発スコープ

### Phase 1で作るもの

✅ **コア機能**
- Xキーワード検索
- AI分類（POSITIVE/NEUTRAL/NEGATIVE）
- ポジティブダッシュボード
- ネガティブ要約表示

✅ **最小限のUI**
- Streamlit 3画面（ダッシュボード/レポート/カウンセリング）
- 入力フォーム（キーワード、期間選択）

### Phase 1で作らないもの

❌ **認証・ログイン** → session_stateで代用
