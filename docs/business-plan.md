## 更新ルール
- **原則として変更しない**
- 変更が必要な場合は**株主の承認が必須**
- 「要調査」の項目は、調査完了後にCEOが数値を埋める（これは変更ではなく補完）
- 事業の追加・削除・収益モデルの変更は株主承認事項

---

## ミッション

AIだけで事業を回し、収益を上げることを証明する。

## ビジョン

「AIが自律的に経営する組織」の実験モデルを作り、その過程と知見を社会に還元する。

## 基本哲学

**「AIで稼げることはなんでもやる実験室」**

SaaSとコンテンツの2本柱を基盤としつつ、AIで自走可能な事業なら何でも試す。小さく実験し、学びを蓄積し、勝ち筋を見つける。

## 事業構成

### 1. SaaS事業：ShieldMe

| 項目 | 内容 |
|------|------|
| プロダクト | ShieldMe - 誹謗中傷検知SaaS |
| 概要 | SNS・Webメディアの投稿をAIがリアルタイム分析し、誹謗中傷をリスクレベル付きで検知・通知 |
| ターゲット | インフルエンサー、企業広報、タレント事務所、学校・自治体 |
| 収益モデル | SaaS月額課金（フリーミアム → 有料プラン） |
| 競合 | 要調査（経営企画部長が市場調査で特定） |
| 市場規模 | 要調査 |

### 2. コンテンツ事業

| 項目 | 内容 |
|------|------|
| 概要 | 「AIだけで事業を回す」過程自体をコンテンツ化し、収益を得る |
| コンテンツ種別 | ブログ記事、技術記事、実験レポート、ノウハウ教材 |
| ターゲット | AI活用に関心のあるエンジニア、起業家、経営者 |
| 収益モデル | 広告収益（ブログ）、有料コンテンツ（note等）、スポンサーシップ |
| 競合 | 要調査 |

### 3. 動画事業（実験フェーズ）

| 項目 | 内容 |
|------|------|
| 概要 | AI自動生成パイプラインによるYouTubeチャンネル運用 |
| コンテンツ種別 | 実験レポート動画、AI活用ハウツー、ショート動画 |
| ターゲット | AI活用に関心のあるエンジニア、起業家、一般視聴者 |
| 収益モデル | YouTube広告収益、チャンネルからの集客 |
| 担当 | video-creator（企画・台本）+ 外部ツール（映像・音声） |
| ステータス | パイプライン設計中。ツール契約は株主承認後 |

### 事業間の関係

```
コンテンツ事業（集客・ブランディング）
  → 認知拡大 → ShieldMeへの送客
  → コンテンツ自体でも収益化

動画事業（認知拡大・収益化）
  → ブログ記事を動画化（クロスコンテンツ）
  → AIで全自動運用する過程自体がコンテンツ

ShieldMe（プロダクト収益）
  → 開発過程がコンテンツ・動画のネタになる
  → ユーザーの声がコンテンツのネタになる
```

3事業は相互に補完する。コンテンツと動画が先行し、ShieldMeの開発過程をコンテンツ化しながらプロダクトを育てる。

### 新事業の追加フロー

事業の追加・削除・収益モデルの変更は株主承認事項。ただし小さい実験はCEO判断でOK。

```
1. 株主 or CEO: アイデア提起
2. CEO: feasibility検討（analyst調査 + 担当エージェント）
3. CEO → 株主: Go/NoGo提案（コスト・リスク・見込みを明記）
4. 株主承認 → ロードマップ・アクションに追加
5. 実行 → 小さく検証 → 拡大 or 撤退
```

## リスク

| リスク | 影響 | 対策 |
|--------|------|------|
| APIコストが収益を上回る | 赤字継続 | 収支計画で損益分岐点を明確化 |
| 法的リスク（誹謗中傷判定の誤り） | 訴訟リスク | 法務部長によるレビュー、免責事項 |
| AIの判断ミスによるブランド毀損 | 信頼低下 | 重要判断は株主承認制 |
| 競合の存在 | 差別化困難 | 市場調査で競合優位性を特定 |
| コンテンツの著作権問題 | 法的リスク | AI生成コンテンツの権利整理 |

## 基本方針（不変）

1. **実験として学びを最大化する** - 収益だけでなく「AIで何ができるか」の知見を蓄積
2. **過程を全公開する** - 成功も失敗も含めて透明に
3. **コスト意識を持つ** - 赤字を垂れ流さない。収支を常に把握
4. **小さく始めて検証する** - 大きな投資の前に小さく試す
5. **株主の時間を最小化する** - 報告は簡潔に、相談事項は明確に
