# Type A事業 コンテンツ戦略分析レポート
## 「AIで事業を作る過程をドキュメンタリーとして発信する」

## 調査日: 2026-02-19
## 調査者: analyst（経営企画部長 / CFO兼務）

---

## サマリー（3行）

日本語ビルドインパブリック系コンテンツは「成功ノウハウ」より「生々しいプロセス」が売れる時代に突入した。AIっぽい文章（AI slop）が検索を汚染しているため、個人の体験・失敗・数字の実測が差別化の核になる。プラットフォーム別に読者層と訴求軸が異なるため、単一コンテンツの焼き直しは通用しない。

### 1. X（Twitter）での発信パターン

#### 代表的アカウント: チャエン（@masahirochaen）
- フォロワー数: 17.7万人（推定。出典: digirise.ai）
- ジャンル: AIニュース速報、生成AI活用術
- 投稿スタイル: 「毎日最速で発信」が核心。速報性を最大の武器にしている
- 投稿頻度: 1日複数回（毎日投稿必須と自社メディアで明言）
- トーン: カジュアル寄り、「⚡️」などの記号を多用、熱量重視
- バズった投稿の型（digirise.ai、2025年公開の分析より）:
  - 【新機能告知型】: 「【速報】〇〇の新機能が来た。これは使える」+ 箇条書き3点
  - 【実用・解説型】: 「劇的にラクになる〇〇の使い方」+ 具体的手順
  - 【週次まとめ型】: 「今週重要なAIニュース5選」+ 画像or図解
- 画像・動画: ほぼ全投稿に画像または動画を添付（テキスト単独は最小化）
- ハッシュタグ: ほぼ使わない（Xアルゴリズムがハッシュタグを評価しない傾向）

#### 代表的アカウント: 深津貴之（@fladdict）
- フォロワー数: 17.8万人（出典: Salesforceブログ）
- ジャンル: UI/UX + 生成AI解説、AIの本質論
- 投稿スタイル: 長文スレッド形式が多い。「わかりやすく噛み砕く」ことに特化
- トーン: 知的、丁寧、専門性を平易な言葉で説明
- バズるパターン: 新しいAIモデルや概念を「誰でも分かるたとえ」で説明する投稿

#### 2026年Xアルゴリズムの現実（出典: X公式ソースコード GitHub xai-org/x-algorithm、2026年1月公開）

| アクション | いいねを1とした時の重み |
|-----------|----------------------|
| リプライへの返信 | 75倍 |
| リポスト（RT） | 約20倍 |
| 引用リポスト | 約15倍 |
| ブックマーク | 約10倍 |
| リプライ | 約13.5倍 |
| いいね | 1倍（基準） |

**示唆（So What?）**:
- 「バズる」= いいねを集めるではない。**リプライが来る投稿を設計する**ことが最重要
- 投稿から6時間でスコアが半減する「時間減衰」があるため、投稿時間（朝7〜9時、昼12〜13時、夜21〜23時）が重要
- 外部URLを含む投稿はアルゴリズムが評価を下げる（出典: 各マーケティングメディア）
- 動画付き投稿はテキストのみより約10倍の反応（出典: comnico.jp分析）

#### 日本語AI系Xアカウントで「刺さる」表現パターン

```
（バズりやすい型 1）: 告白型
「正直に言います。〇〇を3ヶ月やってみた結果が衝撃でした」
→ 失敗・驚き・自己開示。返信が来やすい。

（バズりやすい型 2）: 数字実測型
「Claude APIで100件処理したら費用が〇円だった。計算式と全記録」
→ 誰も調べていない一次データ。引用RTされやすい。

（バズりやすい型 3）: 問題提起型
「AIスタートアップが1年で気づいたこと。誰も教えてくれなかった話」
→ 「自分も知りたい」を引き出す。フォロワー外への拡散が起きやすい。
```

#### 売れているタイトルのパターン

以下は検索上位に出現したタイトルの構造分析（推定。出典は各noteページ）:

- 年号 + 最新版: 「【2026年最新版】〇〇完全ガイド」
- 数字 + 結果: 「月10万円達成した全手順」「100件実測した費用」
- 逆説・意外性: 「文字数は関係ない」「専門家より素人が売れる理由」
- 生々しさ: 「2年間の失敗まとめ」「正直に言う、最初の3ヶ月の実態」

#### 売れるnote記事の構造（実用ノウハウ系）

```
1. 無料エリア（300〜500字）:
   - 「この記事を読むと○○ができるようになる」を最初の1文で宣言
   - 著者の実績・体験（なぜこの人が書けるか）
   - 購入を迷う読者への「損しない理由」

2. 有料エリア:
   - 具体的な手順（ステップ形式）
   - 実測データ・実額・実スクショ
   - 失敗談・陥りやすい罠
   - テンプレート・コピペ素材

3. 重要な原則:
   - 有料エリアの価値は「長さ」ではなく「再現性」
   - 自分の体験に根ざした情報でなければ読者は動かない
```

#### 読まれる記事の共通パターン

1. **体験談 + 数字**: 「〇〇をやってみたら、1ヶ月でこうなった」

2. **独自の観点**: 技術系でZennに出てこないキーワード（「工場」「地方」「非エンジニア」）

- Qiita記事のLike数中央値: 0（出典: mima_ita分析）

- note記事がAI（Perplexity等）の引用元ランキング2位に（出典: note公式、日経）

## 示唆（So What?）

### 1. コンテンツの差別化軸は「生々しさ × 一次データ」

AIスロップ（AI slop）の定義: 「スラスラ読めるのに後から何も残らない」「固有名詞がなく、数字が出てこない、一般論の寄せ集め」（出典: Wikipedia AIスロップ、KDDI biz）。

2026年の読者は**AIっぽさを嗅ぎ分ける**ようになっている。差別化のポイントは以下の3つ：

- **一次データ**: 自分で実測した数字（APIコスト、フォロワー増減、売上実額）
- **失敗の記録**: うまくいかなかったことを具体的に書く（AIには書けない）
- **固有名詞の密度**: 「Claudeのhaiku-4-5で100件処理したら$0.32だった」レベルの具体性

### 2. 「ドキュメンタリー型」はnoteとZennで最も有効

- note: 体験談＋ノウハウのハイブリッド。無料で引き込んで有料で実録公開
- Zenn: 技術+体験のハイブリッド。「非エンジニアがAI事業を作る実況」は希少価値が高い
- ブログSEO: 「AI会社実験 費用 実録」など長尾キーワードで入口を作る

### 3. Xは「リプライが来る投稿」に設計する

いいね狙いの投稿と、リプライ狙いの投稿は設計が異なる。リプライ→返信のループがスコアの75倍爆発を引き起こす。「あなたはどう思いますか？」「自分はこう失敗した、同じ経験ある人いますか？」という問いかけ型の投稿が有効。

### 4. YouTubeは量産ではなく「実験レポート」に特化

2026年のアルゴリズムはAI大量生成コンテンツを評価減。10本の薄い動画より1本の「完視聴される動画」が有効。「仮想機関AIを作る実験、第1話」というドキュメンタリーシリーズ形式が差別化になる。

## CEOへの推奨アクション

1. **X投稿の設計方針**: 「リプライが来る」を最優先設計基準にする。具体的な失敗・実測数字・問いかけを含む投稿を週3回以上を目標とする
2. **note記事のポジショニング**: 実用ノウハウ系（¥1,842相場）で「AIで会社を作る全記録」シリーズとして展開。無料エリアで数字（APIコスト実測等）を見せ、有料エリアで判断プロセスを開示する
3. **Zenn記事の活用**: 「非エンジニアがAI SaaSを1人で作る実況」はZennで希少性が高い。技術系の内容はZennで書き、SEO + はてブ経由の流入を狙う
4. **YouTube**: Phase 1以降、「実験ドキュメント」シリーズとして月1本の長尺動画から始める。サムネイルは「顔出し+数字」が基本型
5. **AIスロップ回避の原則を社内共有**: writer・narrator・x-managerに「固有名詞・実測数字・失敗談必須」のコンテンツルールを徹底させる

## 「AIスロップにならない」差別化チェックリスト

以下の項目を満たすコンテンツのみ公開する:

- [ ] 固有名詞（ツール名・金額・日付・人名）が1段落に1つ以上ある
- [ ] 自分が実際にやったこと・測定した数値が含まれている
- [ ] 「一般的には〜」「場合によっては〜」という表現を使っていない
- [ ] 失敗したこと・予想外だったことが正直に書かれている
- [ ] 最後まで読んだ人が「次に何をすればいいか」が分かる
- [ ] 文章をAIに渡して生成させた後、自分の言葉で30%以上書き直している
