# サイト更新

## 引数
- $ARGUMENTS: 更新内容（例: "料金セクション追加", "FAQ更新", "全体デザイン調整"）
  - 引数なし → CEOが更新内容を決めてから開始

## ワークフロー

### Step 1: 更新指示の整理（CEO）
- 何を変えるか（新セクション追加 / 既存修正 / デザイン変更）
- 対象ファイル（site/index.html が主）
- テキスト内容（writerが書いたコピーがあればそのパス）

### Step 2: 実装（site-builder）
site-builderをTask起動:

```
更新内容: {内容}
対象ファイル: site/index.html（他にあれば追記）
テキスト素材: {あればパス}

以下を必ず守れ:
1. docs/design-rules.md を最初に読んでデザインルールに従え
2. Tailwind CSS v4（CDN）を使用
3. ダークテーマ
4. モバイルファースト（レスポンシブ必須）
5. アクセシビリティ（alt属性、コントラスト比、キーボード操作）
6. 既存のデザインシステムとの一貫性

変更前にブラウザで確認できるようにせよ。
```

### Step 3: 確認（CEO）
- ブラウザで `open site/index.html` して目視確認
- デザインルールに違反していないか
- モバイル表示は崩れていないか

## 出力
変更ファイル: `site/` 配下

## 注意
- サイトは未公開（ドメイン未取得）。ローカルでの確認のみ
- JavaScriptは最小限。静的HTMLを基本とする
