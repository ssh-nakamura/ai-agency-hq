# ブログ記事制作

## 引数
- $ARGUMENTS: テーマまたは指示（例: "AI副業の始め方", "ShieldMeの開発裏話"）
  - 引数なし → CEOがテーマを決めてから開始
  - `--type-b {brand-id}` → Type Bモード（仮想機関AI計画の名前を出さない）

## ワークフロー

### Step 1: 企画（CEO判断）
以下を決める:
- テーマと切り口
- ターゲット読者（誰に向けて書くか）
- Type A（仮想機関AI計画ブランド）or Type B（ニッチブランド）
- 公開先（自社ブログ / note / はてブ）
- 参考にするトレンド（`/trend-research` の結果があれば活用）

### Step 2: 執筆（writer）
writerをTask起動して以下を指示:

```
テーマ: {テーマ}
ターゲット: {読者像}
Type: A / B（Type Bの場合はブランドシート content/type-b/{id}/brand.md を読め）
文字数目安: 2,000〜4,000字
SEOキーワード: {あれば}

以下の構成で書け:
1. フック（最初の3行で読者を引き込む）
2. 本文（見出し3〜5個、具体例を必ず入れる）
3. まとめ（読者の次のアクションを示す）

成果物: content/blog/{slug}.md
```

### Step 3: レビュー（CEO）
- 読者目線で読み直す
- 事実誤認がないか確認
- Type Bの場合: AI制作であることが漏れていないか確認
- OKなら公開指示、NGならwriterに修正指示

### Step 4: 記録
- `docs/status.md` のブログ記事数を更新
- 公開後はURLを記録

## 出力
成果物: `content/blog/{slug}.md`

## 品質基準
- 見出しが3つ以上
- 具体例が1つ以上
- 読者への問いかけまたはCTAがある
- 誤字脱字チェック済み
