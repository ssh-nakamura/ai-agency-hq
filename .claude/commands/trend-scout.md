# Trend Scout — グローバルマーケットインテリジェンス

> プロのマーケリサーチャーが使う水準のトレンド調査スキル。
> 海外↔日本の双方向で市場シグナルを捕捉し、Go-to-Market戦略まで落とし込む。

## 起動

```
/trend-scout                          # インタラクティブモード
/trend-scout 海外→日本 AI content     # 方向 + テーマ指定
/trend-scout 日本→海外 誹謗中傷対策
/trend-scout 双方向 AI株取引
```

CEOがwriterをTask起動して実行する。引数がなければ方向・テーマ・対象事業をCEOに確認してから開始。

## Phase 1: インテリジェンス収集（情報を集める）

### 1-1. ソース別サーチマトリクス

方向ごとに以下のソースを**必ず全部**当たる。最低15クエリ。

#### 海外→日本（海外で何が来ているか）

| ソースカテゴリ | 具体的な調査先 | 検索例 |
|--------------|--------------|--------|
| **スタートアップ動向** | Product Hunt（直近30日のトップ）、Y Combinator batch | "{theme} site:producthunt.com 2026" |
| **VC / 資金調達** | Crunchbase, PitchBook, TechCrunch Funding | "{theme} Series A 2026", "{theme} funding round" |
| **テックメディア** | TechCrunch, The Verge, Ars Technica, Wired | "{theme} trend 2026" |
| **コミュニティの熱量** | Reddit（r/SaaS, r/startups, r/artificial）, Hacker News | "site:reddit.com {theme} growing" |
| **SNSバズ** | X(Twitter)のバイラル投稿、LinkedIn thought leaders | "{theme} viral OR trending" |
| **動画トレンド** | YouTube急上昇、TikTok trending hashtags | "{theme} youtube channel growth 2026" |
| **レポート / データ** | Gartner Hype Cycle, CB Insights, Statista | "{theme} market size forecast" |

#### 日本→海外（日本の何が海外に刺さるか）

| ソースカテゴリ | 具体的な調査先 | 検索例 |
|--------------|--------------|--------|
| **海外から見た日本** | 英語で "Japan {theme}" の評価 | "Japan {theme} innovative OR unique" |
| **日本発の海外成功事例** | 日本企業のグローバル展開ケース | "{テーマ} 海外展開 成功事例 2026" |
| **文化的ユニーク性** | 日本特有で海外に存在しない概念 | "Japanese {theme} no equivalent" |
| **在日外国人の反応** | Reddit r/japan, r/JapanLife | "site:reddit.com japan {theme}" |
| **国内最新動向** | note, Zenn, はてブ, NewsPicks | "{テーマ} 最新 トレンド 2026" |
| **政策・規制** | 経産省, デジタル庁, 総務省 | "{テーマ} 政策 ガイドライン 2026" |

### 1-2. シグナル強度の判定

各発見に対して、以下のシグナル指標を記録:

| シグナル | 測定方法 | 強/中/弱の基準 |
|---------|---------|---------------|
| **検索ボリューム推移** | Google Trends（過去12ヶ月） | 強: +50%以上、中: +20-50%、弱: 横ばい |
| **資金流入** | Crunchbase直近6ヶ月の調達額 | 強: $100M+、中: $10-100M、弱: $10M未満 |
| **コミュニティ規模** | Subreddit登録者 / Discord参加者 | 強: 10万+、中: 1-10万、弱: 1万未満 |
| **メディア露出** | TechCrunch等A級メディアの記事数 | 強: 月5+、中: 月1-5、弱: 月1未満 |
| **プレイヤー数** | 同一領域のスタートアップ数 | 強: 20+（市場成立）、中: 5-20、弱: 5未満（Early） |
| **大手参入** | GAFAM / 大企業の動き | 強: 参入済み、中: 検討中、弱: 未着手 |

## Phase 2: 分析（集めた情報を構造化する）

### 2-1. トレンドライフサイクル判定

```
  Innovation          Early            Growth           Maturity          Decline
  Trigger            Adopters          Phase            Phase             Phase
     │                  │                │                │                │
     ▼                  ▼                ▼                ▼                ▼
  ┌─────┐          ┌─────┐          ┌─────────┐      ┌─────┐          ┌─────┐
  │ 発見 │──────────│ 検証 │──────────│ スケール │──────│ 飽和 │──────────│ 衰退 │
  └─────┘          └─────┘          └─────────┘      └─────┘          └─────┘
   狙い目 ◎          狙い目 ◎          参入可 ○         遅い △           撤退 ✕
```

各トレンドをこの曲線上にマッピングする。**狙うのはInnovation Trigger〜Early Adopters。**

### 2-2. ローカライゼーション適合度マトリクス（CLPR分析）

海外↔日本の移植可能性を4軸で評価（各1-5点、合計20点満点）:

| 軸 | 評価内容 | 5点（容易） | 1点（困難） |
|-----|---------|-----------|-----------|
| **C**ulture（文化） | 文化的に受容されるか | 普遍的ニーズ | 文化依存が強い |
| **L**anguage（言語） | 翻訳・ローカライズの難易度 | 技術用語中心 | ニュアンス依存 |
| **P**latform（流通） | 配信チャネルが存在するか | 同一PF（YouTube等） | PFが異なる |
| **R**egulation（規制） | 法規制の障壁 | 規制なし | 要許認可 |

**16点以上: 即展開可能 / 12-15点: 調整必要 / 11点以下: 要慎重検討**

### 2-3. 競合インテリジェンス

各トレンド領域で以下を調査:

```
■ 直接競合（同じ課題を同じ方法で解決）
  → 名前、URL、資金調達額、ユーザー数、価格帯

■ 間接競合（同じ課題を別の方法で解決）
  → 代替手段、ユーザーの現状の対処法

■ 空白地帯（誰も狙っていないポジション）
  → なぜ空白なのか（市場がない？技術がない？気づいていない？）

■ 日本 vs 海外の競争環境差
  → 海外は激戦だが日本は空白、またはその逆
```

## Phase 3: 戦略化（So Whatを出す）

### 3-1. Go-to-Market スコアリング

各トレンドに対して、事業への適用可能性をスコアリング:

| 評価基準 | 重み | 1点 | 3点 | 5点 |
|---------|------|-----|-----|-----|
| 事業インパクト | ×3 | 間接的 | 1事業に効く | 複数事業に波及 |
| 実行コスト | ×2 | ¥50,000+/月 | ¥10,000-50,000/月 | ¥10,000未満/月 |
| 時間軸 | ×2 | 6ヶ月以上 | 1-3ヶ月 | 今月着手可 |
| 先行者利益 | ×2 | 既に多数参入 | 数社が先行 | 日本で未開拓 |
| コンテンツ価値 | ×1 | 記事にしにくい | 1本記事が書ける | シリーズ化できる |

**合計30点満点。20点以上: 即着手推奨 / 15-19点: 検討価値あり / 14点以下: 見送り**

### 3-2. アクションプラン生成

スコア上位3件について、具体的なGTMプランを記述:

```
■ トレンド名: {名前}
■ GTMスコア: {X}/30
■ 対象事業: SaaS / コンテンツ / 動画 / 株取引
■ ターゲット: {誰に、どの市場で}
■ ポジショニング: {既存プレイヤーとどう差別化するか}
■ 最初の一手: {最小コストで検証できるアクション}
■ 成功指標: {何が起きたら成功と判断するか}
■ 撤退基準: {何が起きたらやめるか}
■ 必要リソース: {エージェント、予算、期間}
```

## 定期実行

| 頻度 | スコープ | 目的 |
|------|---------|------|
| **月1回** | 全事業 × 双方向（フルスキャン） | 新トレンドの発見、既存監視項目の更新 |
| **随時** | 特定テーマの深掘り | 新事業検討時、競合の動きがあった時 |
| **四半期** | GTMスコア上位の再評価 | 参入判断の見直し |

## 品質基準

- 検索クエリ数: **最低15。20以上が望ましい**
- ソース言語: **英語と日本語の両方を必ず含める**
- トレンド件数: **最低5件を発見。上位3件にGTMプラン**
- バイアス開示: **自分の視点の限界を必ず明記**
- 再現性: **全検索クエリをレポートに記録**
