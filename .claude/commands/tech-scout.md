# 技術スカウト

定期的にClaude Code周辺の新技術を調査し、使えるものを導入する。

## 1. 調査対象（Web検索で確認）

以下のカテゴリを検索する:

### Claude Code ツール
- 新しいCLIツール（ccusage等の分析系）
- GUIクライアント（opcode, Claudia, CodePilot等）
- ダッシュボード（ccboard等）
- ログビューア（clog, claude-run等）

### MCP サーバー
- 公式MCP（Anthropic提供）
- コミュニティMCP（GitHub trending）
- 業務に使えそうなMCP（ファイル操作、API連携、DB等）

### プラグイン・拡張
- Claude Code plugins（claude-mem等）
- VS Code拡張
- フック（hooks）の活用事例

### 競合・代替ツール
- Cursor, Windsurf, Cline等の動向
- 他AIコーディングツールの新機能
- 取り入れられるアイデア

## 2. 評価フレームワーク

各ツールを以下で評価:

| 評価軸 | 基準 |
|--------|------|
| 必要性 | 今の課題を解決するか？（High/Medium/Low） |
| コスト | 無料 or 有料。有料なら月額いくらか |
| 導入難易度 | npm/brew一発 or 設定が複雑か |
| ライセンス | MIT/Apache → OK、GPL → 要注意 |
| 成熟度 | Star数、最終更新日、Issue対応状況 |
| トークン効率 | トークン消費を増やすか減らすか |

**導入判断**:
- 無料 + 必要性High + 導入容易 → CEO判断で即導入
- 有料 or 必要性Medium → CEO検討、必要なら株主承認
- GPL or リスクあり → legal確認後にCEO判断

## 3. 導入手順

1. ライセンス確認 → `docs/legal/oss-licenses.md` に記録
2. テスト導入（ローカル）
3. 動作確認
4. エージェント定義に追加（必要なら `.claude/agents/*.md` 更新）
5. スキルに追加（必要なら `.claude/commands/*.md` 作成）
6. CEO MEMORYに記録

## 4. エージェント教育

新ツール導入時:
- 該当エージェントの定義ファイル（`.claude/agents/*.md`）にツール情報を追記
- 使い方の例を記載
- 初回はCEOが使い方を指示に含める

`docs/specs/tech-scout-YYYY-MM.md` に以下を記録:

```markdown
|---------|---------|------|------|

## 導入したもの
-

## 見送ったもの（理由付き）
-

## 次回の注目ポイント
-
```

## 6. 実施頻度
- **月1回**を推奨（月初のセッションで実施）
- 株主から新ツール情報があった場合は随時
- エージェントが作業中に便利そうなツールを見つけたらCEOに報告 → 次回tech-scoutで評価

## 注意
- 調査は実施するが、**導入はCEO判断**（勝手に入れない）
- 有料ツールは必ず株主承認
- 「便利そう」だけで入れない。**今の課題を解決するか**が基準
