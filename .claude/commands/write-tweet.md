# X投稿案作成

## 引数
- $ARGUMENTS: テーマまたは指示（例: "ブログ記事の告知", "今日の学び"）
  - 引数なし → CEOがテーマを決めてから開始
  - `--thread` → スレッド形式（複数ツイート）
  - `--type-b {brand-id}` → Type Bモード

## ワークフロー

### Step 1: 企画（CEO判断）
- 投稿の目的（認知拡大 / エンゲージメント / 告知 / ブランディング）
- Type A or Type B
- 単発 or スレッド

### Step 2: 案作成（x-manager）
x-managerをTask起動して以下を指示:

```
目的: {目的}
テーマ: {テーマ}
Type: A / B
形式: 単発 / スレッド（{N}ツイート）

3案作れ。各案に以下を含めること:
- 本文（280字以内 / 日本語は140字目安）
- 想定エンゲージメント（誰が反応するか）
- 投稿タイミング（曜日・時間帯の推奨）
- ハッシュタグ（2〜3個）

成果物: content/tweets/{YYYY-MM-DD}-{slug}.md
```

### Step 3: 承認（CEO）
- 3案から1案を選択、または修正指示
- Type Bの場合: ブランドの一貫性を確認
- OKなら投稿準備完了

### Step 4: 記録
成果物: `content/tweets/{YYYY-MM-DD}-{slug}.md`

## 注意
- 実際の投稿はX APIまたは手動。このスキルは案の作成まで
- 炎上リスクのある内容はlegalレビューを挟む
