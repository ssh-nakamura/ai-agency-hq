# 市場調査

## 引数
- $ARGUMENTS: 調査対象（例: "日本のAI SaaS市場", "誹謗中傷対策ツール競合"）
  - 引数なし → CEOが対象を決めてから開始

## ワークフロー

### Step 1: 調査スコープ定義（CEO）
- 対象市場（国・地域・業界）
- 調査の目的（参入判断 / 価格設定 / 競合把握 / 投資家向け）
- 深さ（概要スキャン / 詳細分析）

### Step 2: 調査実行（analyst）
analystをTask起動:

```
対象: {対象市場}
目的: {目的}
深さ: {概要 / 詳細}

以下を調査せよ:
1. 市場規模（TAM/SAM/SOM、可能なら数値）
2. 成長率（過去3年 + 今後3年の予測）
3. 主要プレイヤー（名前、シェア、価格帯、強み弱み）
4. 参入障壁（技術、資金、規制、ネットワーク効果）
5. 顧客セグメント（誰が買うか、なぜ買うか）
6. トレンド（何が変化しているか）

成果物: docs/specs/{slug}-market-report.md
```

### Step 3: レビュー（CEO）
- 数値の出典は明記されているか
- 推定値と実績値は区別されているか
- 競合の見落としはないか

## 出力
成果物: `docs/specs/{slug}-market-report.md`

## 品質基準
- 数値には必ず出典（URL or レポート名）を付ける
- 推定値には「推定」と明記
- 3社以上の競合を比較
