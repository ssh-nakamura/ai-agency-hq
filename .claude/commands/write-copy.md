# コピーライティング

## 引数
- $ARGUMENTS: 対象と用途（例: "ShieldMe LP キャッチコピー", "note記事の導入文"）
  - 引数なし → CEOが対象を決めてから開始

## ワークフロー

### Step 1: ブリーフ作成（CEO）
- 対象（LP / 広告 / プロダクト説明 / メール / SNSプロフィール）
- ターゲット（誰に刺さるべきか）
- トーン（プロフェッショナル / カジュアル / 緊急感 / 信頼感）
- 必須キーワード（あれば）
- 既存コピー（リライトの場合）

### Step 2: 執筆（writer）
writerをTask起動:

```
対象: {対象}
ターゲット: {ターゲット}
トーン: {トーン}
キーワード: {キーワード}

3パターン作れ:
- A案: {方向性1}
- B案: {方向性2}
- C案: {方向性3}

各案にキャッチコピー + サブコピー + 本文を含めること。
成果物: content/copy/{slug}.md
```

### Step 3: 選択・修正（CEO）
3案から選択 or 組み合わせ。必要ならwriterに修正指示。

## 出力
成果物: `content/copy/{slug}.md`
