## CEO所見

### 良い点
- YouTubeに明確な需要（JP 573万再生/top20）
- AI×雑学ショートの商業エコシステムが既にJPで成熟
- 情報商材が売れている = マネタイズ実績あり
- 非属人（顔出し不要）で参入障壁が低い

### 懸念点
- **先行者が多すぎる**。@kaneki_ai888、@Kirara_A1、@tassy_youtube等が既にポジション確立済み
- **JPが先行市場**。Trend Arbitrage（EN先行→JP後追い）パターンに該当しない
- **発信者5万人超え**。レッドオーシャン
- Brain/noteの情報商材が既に出回っている = 「教える系」も飽和気味
- 差別化が難しい（AIで雑学台本を量産するのは誰でもできる）

### Type B事業としての判断
**保留**。需要はあるが飽和度が高く、先行者優位を取れない。
Trend Arbitrageパターン（海外先行→JP空白）に該当しないため、Type B戦略の優先度は低い。

ただし「雑学ショート動画の作り方」を教える側（メタコンテンツ）ではなく、
特定ジャンルの雑学（例: 科学雑学、歴史雑学、食の雑学）に特化すれば差別化の余地はある。
→ ジャンル特化の再調査が必要であれば別途実施。

## データ品質メモ（スクリプト化への知見）

1. **Twitter countTweets**: 完全一致フレーズは使えない（0-4件）。broad queryのみ実用的
2. **Twitter posts broad search**: OR一致で10万件超のノイズ。スクリプトではフィルタリングロジックが必須
3. **Reddit JP**: 中国語混入問題。言語フィルターが必要
4. **Instagram**: AIツール宣伝スパムが支配的。商業シグナル検出には使えるが需要測定には不向き
5. **YouTube (yt-dlp)**: 最もクリーンで信頼性の高い需要シグナル。スクリプトの主軸にすべき
6. **Grok API qualitative search**: ナレッジギャップと商業シグナルの定性分析に最適。数値データ収集には不向き

**推奨スクリプト設計**:
- Step 1主軸: yt-dlp (YouTube再生数)
- Step 2-4: Xpoz Twitter (publishers数) + yt-dlp (再生数/チャンネル数)
- Step 3,7: Grok API (定性分析 — スクリプト化しにくい。CEOセッションで判断)
- Step 5-6: 計算のみ（スクリプト内で完結）
